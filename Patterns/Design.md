# Design

## Invariant
Choose the internal representation whose costs match the operations the class must support, then keep every public method consistent with the class invariant. The "algorithm" is the data-structure choice.

## Trigger phrases
- "implement a class that supports the following methods"
- "design a structure for …" (most-recently-used, editor, scheduler)
- operations have stated complexity targets
- mutable state accessed through a small API
